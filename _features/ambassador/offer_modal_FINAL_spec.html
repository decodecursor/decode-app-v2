Offer modal spec — final

Per-professional promotion shown in a coupon modal, opened by the gift icon on the listing card. Book Now opens the professional's WhatsApp with a pre-filled, ambassador-attributed message. Replaces the earlier redemption-code/QR concept.

Status Final · locked Marker Banner · black chip Surface Public page · 360px modal Source offer_modal_FINAL_B.html

Layout & behaviour

Reads top→bottom: what it is (service) → what it costs (struck original above special) → how much off (auto chip) → bonus → expiry → book. The “Promotion” banner makes the nature unmistakable; the black chip reads as a deal stamp.

Anatomy

  1. Hero — category eyebrow (pink) / pro name / city. Shared with the other modals.
  2. Promotion banner — full-width pink strip, white text, top of the ticket.
  3. Service — what the offer is for (e.g. “Haircut”), grey, above the price.
  4. Prices — struck original above, special price below (white, large).
  5. Discount chip — auto-computed “% OFF”, black text on pink.
  6. Perk — gift icon + “Including free / {perk}”, centered, no box. Hidden if empty.
  7. Expiry — “Valid until {date}”, grey.
  8. Book Now — WhatsApp icon; opens the pro's WhatsApp (pre-text below).
  9. Cancel — closes.

Book Now — WhatsApp

Target
the pro's whatsapp_number
Attribution
the page's ambassador first name (the profile whose page this is)
Pre-text
“Hi, I'm interested in {ambassador}'s {service} promotion for AED {special_price}💕”
Example
“Hi, I'm interested in Yanni's Haircut promotion for AED 174💕”
Helper
reuse the existing WhatsApp link + URL-encoding used by the listing-card badge

States & rules

offer showsOnly when the pro has an active offer AND a non-null whatsapp_number. No WhatsApp → no gift icon → no modal.
pricesBoth original_price and special_price are always required; the chip always computes from them.
roundingChip % = Math.round to nearest whole number.
perkHide the perk row (icon + text) when perk is empty.
expiryRequired — always shown.
